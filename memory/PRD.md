# BarberBook - Product Requirements Document

## Overview
Mobile application (Expo React Native) for barbershop appointment scheduling. Barbers share their QR code with clients so they can book appointments seamlessly.

## User Roles
1. **Admin (Dono)** — Manages services and barbers
2. **Barber (Barbeiro)** — Views schedule, shares QR code
3. **Client (Cliente)** — Books appointments, views history

## Core Features Implemented (MVP)
- ✅ JWT authentication (email/password)
- ✅ 3 role types with role-based access control
- ✅ Admin dashboard with stats (bookings, services, barbers)
- ✅ Service management (CRUD) — admin only
- ✅ Barber management with auto-generated QR codes
- ✅ Full booking flow: service → barber → date → time → payment → confirm
- ✅ Available slots calculation (9AM-6PM, 30min intervals)
- ✅ Booking history for clients
- ✅ Barber schedule view (today + upcoming)
- ✅ Booking status: confirmed/completed/cancelled
- ✅ Payment options: pré-pago (MOCK) or no local
- ✅ QR Code sharing (base64 PNG generated by backend)
- ✅ Manual barber code entry (fallback for QR scan)

## Backend (FastAPI + MongoDB)
- `POST /api/auth/register` — Create account
- `POST /api/auth/login` — Login
- `GET /api/auth/me` — Current user
- `POST/GET/DELETE /api/services` — Service CRUD
- `POST/GET /api/barbers`, `GET /api/barbers/{id}` — Barber management
- `POST/GET /api/bookings`, `PATCH /api/bookings/{id}/status` — Booking flow
- `GET /api/bookings/available-slots` — Time slot availability
